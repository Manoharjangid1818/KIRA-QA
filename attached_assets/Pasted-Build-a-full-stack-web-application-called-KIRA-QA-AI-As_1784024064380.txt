Build a full-stack web application called KIRA QA AI Assistant.

KIRA stands for KiwiQA Intelligent Retrieval Assistant.

The application should be an AI assistant for software testing and QA work.

Main Goal

Create a working MVP where a QA engineer can:

Register and log in.
Open a ChatGPT-style AI chat.
Analyze software requirements.
Generate test scenarios.
Generate detailed test cases.
Generate bug reports.
Save generated results.

AI Integration

Do not train an LLM.

Create a configurable AI service that can connect to any OpenAI-compatible API.

Later I will connect this application to a private Qwen3 model running through vLLM.

Use environment variables:

LLM_BASE_URL=
LLM_API_KEY=
LLM_MODEL=

Do not hard-code the model or API URL.

If no real LLM is configured, create a mock AI provider for development.

Important AI Rules

The AI must:

Never claim that generated test cases were actually executed.
Never claim that a bug is confirmed without evidence.
Clearly mention assumptions.
Never invent project information.
Recommend human review for important outputs.
