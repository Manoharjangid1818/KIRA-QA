Upgrade the existing **KIRA AI Assistant** and clearly separate the application into two different interfaces:

1. **Admin Panel**
2. **Employee/User Panel**

Do not rebuild the application from scratch.

Keep all existing functionality:

* Authentication
* Qwen3 integration
* vLLM integration
* Vision model integration
* RAG
* Knowledge Bases
* File uploads
* Image uploads
* Conversation history
* Database
* Existing frontend and backend

The system should automatically open the correct interface based on the authenticated user's role.

# MAIN ROLE FLOW

```text
Login
↓
Authenticate User
↓
Check Role
├── Super Admin / Admin → Admin Panel
└── Employee / User → User Chat Panel
```

Do not allow users to manually select whether they want the Admin or User interface.

The role must come from the authenticated user record in the database.

---

# PART 1 — ADMIN PANEL

The Admin Panel should contain all management, configuration, security, and monitoring features.

Admin route:

```text
/admin
```

Only these roles can access it:

* Super Admin
* Admin

Normal employees must never see or access the Admin Panel.

## Admin Sidebar

Create these sections:

### 1. Dashboard

Show:

* Total Users
* Active Users
* Total Conversations
* Total AI Requests
* Total Knowledge Bases
* Total Documents
* Total Projects
* Total Departments
* Failed AI Requests
* Storage Usage
* System Status

Also show:

* Recent user activity
* Recent document uploads
* Recent errors
* Recent knowledge base updates

### 7. Access Control

Create a separate Access Control section.

Admin should control:

* Which user can access which department
* Which user can access which project
* Which user can access which knowledge base
* Which user can upload files
* Which user can use image analysis
* Which user can access company-wide knowledge
* Which user can use restricted knowledge

Access control must be enforced in the backend.

Do not rely only on hiding frontend menus.


---

### Search Company Knowledge

Employees should simply ask questions.

Example:

```text
What is our QA defect reporting process?
```

The backend should automatically:

```text
Check User
↓
Check Department
↓
Check Assigned Projects
↓
Search Allowed Knowledge Bases
↓
Retrieve Relevant Information
↓
Send Context to Qwen3
↓
Return Answer
```

The user should not manually configure RAG.

# AUTOMATIC KNOWLEDGE ACCESS

Normal users should not see technical knowledge base settings.

The backend should automatically search:

```text
Company Knowledge
+
Department Knowledge
+
Assigned Project Knowledge
```

Only if the user has permission.

Example:

```text
QA Employee
→ Company Knowledge
→ QA Knowledge
→ Assigned Project Knowledge
```

A QA employee must not automatically access HR-restricted knowledge.
