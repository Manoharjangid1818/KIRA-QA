Update the existing **KIRA AI Assistant** into a simple application with only two separate panels:

1. **Admin Panel**
2. **User Chat Panel**

Do not rebuild the application from scratch.

Keep the existing:

* Authentication
* Database
* Qwen3 integration
* vLLM integration
* RAG system
* Vision model integration
* File upload
* Image upload
* Conversation history
* User permissions

Remove all unnecessary, duplicate, old, or unused frontend pages.

# Main Login Flow

Use one common login page.

After login, check the user role from the database.

```text
Login
↓
Check User Role
├── Admin / Super Admin → Admin Panel
└── Employee / Normal User → User Chat Panel
```

Do not show a role-selection option on the login page.

The backend must verify the user role.

---

# PANEL 1 — ADMIN PANEL

Route:

```text
/admin
```

Only Admin and Super Admin users can access this panel.

The Admin Panel should contain only application management options.

Do not add normal chat features or unnecessary QA tools to the Admin Panel.

Keep only these options:

* Dashboard
* Users

# PANEL 2 — USER CHAT PANEL

Route:

```text
/chat
```

Normal users should see a simple interface similar to ChatGPT or Gemini.

Do not show any Admin, AI configuration, RAG, database, model, document-management, or permission-management options.

## User Sidebar

Keep only:

* New Chat
* Search Chats
* Recent Chats
* My Files
* Profile
* Logout

## New Chat

Create a new conversation.

## Search Chats

Allow users to search their previous conversations.

## Recent Chats

Show only the current user's conversations.

Allow the user to:

* Open a chat
* Rename a chat
* Delete a chat
* Pin important chats
* Archive chats

Automatically create a conversation title from the first message.

## Main Chat Screen

Create a ChatGPT-style chat interface with:

* User messages
* AI responses
* Message input box
* Send button
* Attachment button
* Stop response button

* Regenerate response
* Edit and resend user message
* Thumbs up and thumbs down feedback

* Code blocks
* Tables
* Lists
* Streaming responses
* Source references
* File preview
* Image preview

## Automatic Backend Routing

The user should not manually choose the AI model.

The backend should route automatically:

```text
Text Question
→ Qwen3

Document
→ Text Extraction
→ Qwen3

Image or Screenshot
→ Vision Model

Company Knowledge Question
→ Permission Check
→ RAG
→ Qwen3

Document and Image Together
→ Process Both
→ Generate Combined Answer
```

## Company Knowledge Access

When a user asks a company-related question, automatically search only the knowledge sources they are authorized to access.

```text
User Question
↓
Check User Identity
↓
Check Department
↓
Check Assigned Projects
↓
Search Allowed Knowledge Bases
↓
Retrieve Relevant Information
↓
Send Context to Qwen3
↓
Show Answer and Sources
```

A user must never access another department's, project's, or restricted knowledge without permission.

## Temporary Chat Files

Files uploaded by users inside chat should remain temporary conversation files.

They must not automatically become permanent Knowledge Base documents.

```text
User Chat Attachment
=
Temporary and user-specific

Admin Knowledge Base Document
=
Permanent approved company knowledge
```

Show only files uploaded by the current user.

Allow:

* View file name

* View upload date
* Open related conversation
* Delete file if permitted

## User Profile

Allow users to:

* View name
* View email
* View role
* View department
* View assigned projects
* Change password
* Logout

# Security Requirements

The application must:

* Verify roles in the backend
* Protect all Admin routes
* Prevent users from opening `/admin`
* Prevent users from accessing another user's chats
* Prevent users from accessing another user's files
* Prevent unauthorized Knowledge Base access
* Never expose system prompts
* Never expose API keys
* Never expose internal model settings
* Never execute uploaded files
* Validate file extension, MIME type, and size
* Store passwords securely

If a normal user tries to open an Admin route:

```text
Access Denied
```

Then redirect them to:

```text
/chat
```
