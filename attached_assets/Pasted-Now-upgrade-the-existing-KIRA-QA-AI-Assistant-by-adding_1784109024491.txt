## Goal

Allow users to upload approved documents and ask questions based on those documents.

The flow should be:

Document Upload
→ Extract Text
→ Clean Text
→ Split into Chunks
→ Generate Embeddings
→ Store Embeddings
→ User Asks Question
→ Search Relevant Chunks
→ Send Retrieved Context to Qwen3
→ Generate Final Answer

## Text Processing

When a document is uploaded:

1. Extract the text.
2. Remove unnecessary empty spaces.
3. Split the text into smaller chunks.
4. Add some overlap between chunks.
5. Generate embeddings.
6. Store the chunks and embeddings.

Keep chunk size and overlap configurable.

Use environment variables:

RAG_CHUNK_SIZE=
RAG_CHUNK_OVERLAP=
RAG_TOP_K=

## Embedding Model

Create a reusable embedding service.

Do not hard-code the embedding model.

Use:

EMBEDDING_PROVIDER=
EMBEDDING_MODEL=

Keep the architecture flexible so the embedding model can be changed later.

## Vector Search

Use PostgreSQL with pgvector if available.

* Milvus

without changing the main application.

## RAG Question Flow

When the user asks a question:

1. Convert the question into an embedding.
2. Search for the most relevant document chunks.
3. Retrieve the top matching chunks.
4. Add the retrieved text to the prompt.
5. Send the final prompt to Qwen3 through the existing AI service.
6. Return the answer.

The prompt should clearly tell the AI:

"Answer using the provided document context.

Do not invent information that is not available in the provided context.

If the answer cannot be found in the provided documents, clearly say:

'I could not find enough information in the available documents to answer this question.'

Use general knowledge only when explicitly allowed."

Users must only access knowledge bases they are authorized to use.

Do not rely only on prompts for access control.

Apply permission filtering before:

* Viewing a knowledge base
* Viewing documents
* Searching embeddings
* Retrieving document chunks
* Sending context to Qwen3

A user must never retrieve content from an unauthorized knowledge base.

## Important Rule

The RAG system should help reduce hallucinations, but the AI must still clearly mention when information is missing.

Do not claim that a document contains information unless that information was actually retrieved.

Build this as an extension of the existing KIRA application without breaking the current features.
